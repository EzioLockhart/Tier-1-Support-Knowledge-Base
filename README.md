## 📝 Article Format

Every troubleshooting guide in this repository follows a consistent 5-section structure:

1. **Symptoms**
   - What the user reports seeing or experiencing

2. **Quick Checks (30 seconds)**
   - Rapid preliminary checks before diving deeper

3. **Step-by-Step Resolution**
   - Numbered, clear instructions anyone can follow

4. **Root Cause**
   - The underlying cause of the issue (when identifiable)

5. **Escalation Criteria**
   - Clear triggers for when to escalate to Tier 2

**Why this format?**
- Consistency speeds up information retrieval during live calls
- Non-technical users can follow "Step-by-Step Resolution" independently
- Escalation criteria prevent wasted time on issues beyond Tier 1 scope
- Clear documentation reduces repeat tickets for the same issue

## 🔧 How To Use This Knowledge Base

### For Support Agents (Simulated Workflow)
1. Identify the issue category (Networking, Email, Printers, etc.)
2. Navigate to the appropriate folder
3. Open the relevant troubleshooting article
4. Follow the **Quick Checks** first, then **Step-by-Step Resolution**
5. Escalate to Tier 2 if the issue meets the defined **Escalation Criteria**

### For Recruiters & Hiring Managers
1. Browse the folder structure to see how information is organized
2. Open 2-3 articles to evaluate technical writing clarity
3. Check the commit history to see version control practices
4. Review this README for project scope and methodology

### For Fellow Learners
1. Clone this repository to use as a template
2. Fork and customize with your own articles
3. Use the article format as a starting point for your own KB
4. Star the repo if you find it useful
